WHAT WE FIXED
================================================================================

1. ✅ GROUPED VERSES DESERIALIZATION (FIXED)
   --------------------------------------------------------------------------------
   Problem: API returns some verses as grouped arrays (e.g., verses 4-6 together)
   - Example: /tel/verse/1/5 returns verse_no: [4,5,6] and verse: [text1, text2, text3]
   - Old deserializers only saved first verse, losing verses 5 and 6
   - This caused 62 verses to be missing during initial download

   Solution:
   - Created GitaVerseRaw.kt to preserve array structures from API
   - Added toIndividualVerses() method to split grouped responses
   - Updated download logic to save each verse individually
   - Fixed duplicate insertion error handling

   Files Modified:
   - Created: data/GitaVerseRaw.kt
   - Modified: repository/OfflineCacheRepository.kt (added raw endpoint handling)
   - Modified: network/GitaApiService.kt (added getVerseRaw() method)

2. ✅ NULLABLE BOOLEAN COMPILATION ERROR (FIXED)
   --------------------------------------------------------------------------------
   Problem: Kotlin doesn't allow !nullable?.contains() == true pattern

   Solution:
   - Changed from: if (!e.message?.contains("UNIQUE constraint") == true)
   - Changed to: if (e.message?.contains("UNIQUE constraint") != true)

   File: repository/OfflineCacheRepository.kt (lines 85, 152)

3. ✅ API ENDPOINT VERIFICATION (COMPLETED)
   --------------------------------------------------------------------------------
   Verified: Out of 701 verses, only 8 are missing from Telugu API
   - Missing: Chapter 9, verses 23-30 (API limitation, not app issue)
   - Available: 693 verses (98.86% coverage)
   - Result saved: missing_verses.txt

4. ✅ WIDGET INTERNET SUPPORT (ADDED)
   --------------------------------------------------------------------------------
   - Widget now tries internet first if network available
   - Falls back to cache if offline or fetch fails

   File: widget/GitaVerseWidget.kt

5. ✅ UI IMPROVEMENTS (COMPLETED)
   --------------------------------------------------------------------------------
   - Unified duplicate progress indicators into single card
   - Shows: "$cachedCount / 700 verses cached"
   - Progress bar appears only during download
   - Dynamic title based on status

   File: ui/OfflineDownloadScreen.kt

CURRENT DOWNLOAD LOGIC FLOW
================================================================================

File: repository/OfflineCacheRepository.kt

fun downloadAllVerses(): Flow<DownloadProgress> = flow {
    1. Get initial cached count
    2. Emit starting progress
    3. Load all cached verse IDs into Set (for fast lookup)
    4. FIRST PASS: Download verses 1-18 by chapter/verse
       - For each chapter (1-18):
         - For each verse (1-maxVerses):
           - Check if already cached (using Set)
           - If not cached:
             - Fetch using getVerseRaw()
             - Split into individual verses
             - Save each verse to database
             - Update progress
             - Delay 50ms (rate limiting)
           - On error: add to failedVerses list
    5. SECOND PASS: Retry failed verses (up to 3 attempts)
       - For each failed verse:
         - Retry with longer delays (1s, 2s, 3s)
         - Update success/failure counts
    6. Emit final completion status

Issues with Current Logic:
- No proper error recovery mechanism
- May not handle database errors gracefully
- Progress updates might block download
- No cancellation support
- No resume capability if interrupted
